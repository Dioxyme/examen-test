L'architecture de cette chaîne repose sur une automatisation en partant du code source jusqu'au déploiement sur AWS via une instance EC2. Le cycle commence par le push du code Node.js 18 sur le dépôt, puis la phase build de l'image Docker et le push de cette dernière sur DockerHub. En parallèle, l'infrastructure est pilotée par Terraform qui va créer l'instance EC2, définir le Security Group (ports 22 et 8080) et gérer les clés d'accès. Enfin, on se connectant à l'instance via SSH pour mettre à jour les packages, récupérer notre image Docker et lancer le conteneur.
